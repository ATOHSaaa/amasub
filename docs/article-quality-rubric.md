# 記事品質スコアリング・リライトルール（アマサブ）

記事の品質評価とリライト時の判断基準。SEO執筆の詳細は [seo-article-guide.md](./seo-article-guide.md) を正とし、本ファイルは**採点・改善の運用**を補う。

**最終更新:** 2026-06-01（全38記事・SEO採点90点以上・KU表記除去）

---

## 1. 100点満点の内訳

| 観点 | 配点 | **90点以上**の目安 |
|------|------|-------------------|
| SEOメタ・KW | 20 | title前半にKW、description 120〜160字、slugが意味を持つ、初段80〜150字で結論 |
| 検索意図・構成 | 25 | ガイド推奨構成を満たす。H2が4〜8本（最大14）で役割が重複しない |
| 深さ・網羅 | 20 | 比較・活用 **5,000字以上**、作品別 **4,500字以上**（フロントマター除く本文） |
| E-E-A-T | 15 | 変動注意・公式確認・断定しすぎない |
| 回遊・独自性 | 10 | 内部リンク **3本以上**、FAQまたは勘違い表 |
| メディア適合 | 10 | 記事タイプ優先度、`services` / `cta` の一致 |

**運用目標:** 公開記事はすべて **90点以上** を維持する。下回ったらリライト（最大3回）。

---

## 2. リライトで点数が上がった要因

### 2.1 比較・柱記事（例: ku-vs-audible）

| 要因 | 具体策 |
|------|--------|
| 結論ファースト | 初段で選び方＋月額 |
| 厚み | 料金表＋違い＋端末＋勘違い＋早見表＋確認手順 |
| 内部リンク | **3〜4本** |

### 2.2 作品別記事

| 要因 | 具体策 |
|------|--------|
| 構成完走 | 手順 → 魅力 → 代替表 → FAQ → まとめ |
| 内部リンク | 3〜4本 |

### 2.3 活用法・ジャンル記事

| 要因 | 具体策 |
|------|--------|
| **FAQ（H2）** | ロングテールKWを4問程度まとめる |
| **勘違い表** | 比較・選び方記事で「思い込み vs 実際」を表にする |
| **確認手順の独立H2** | 活用法でも番号リストを明示（E-E-A-T） |
| **description** | 120〜160字、KW＋記事の独自要素 |
| **内部リンク3本以上** | ハブ（ebook-subscription-guide、ku-vs-audible）と横の記事 |

---

## 3. リライト前チェック（自己採点）

- [ ] 初段に**結論**と**公式で確認**の一文
- [ ] メインKWが title / description / 初段 / H2にある
- [ ] 作品別: 手順・代替・FAQ
- [ ] 比較・活用: 表＋FAQまたは勘違い表
- [ ] 内部リンク **3本以上**
- [ ] 本文 **5,000字以上**（作品別4,500字以上）
- [ ] `services` と `cta` 一致
- [ ] 本文・メタに **「KU」略称を使わない**（Kindle Unlimited と表記）
